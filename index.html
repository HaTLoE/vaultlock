Process

Three steps.
Fully private.

Everything happens inside your browser tab. The moment you close the tab, all keys and file data are gone from memory — permanently.

Step 01
📂

Select your files

Drag and drop one or more files of any type onto the drop zone. Multiple files are bundled into a single archive before encryption — you get one .vault file back.

Step 02
🗝️

Copy your password

A unique 256-bit key is generated in your browser using a cryptographically secure random number generator. Copy it somewhere safe — it is never stored anywhere.

Step 03
⬇

Download your vault

Click Encrypt. Your files are compressed with DEFLATE, encrypted with AES-256-GCM, and packaged into a .vault file — ready to share safely over any channel.

FAQ

Common questions

Does anything get uploaded to a server? +
No. Encryption and decryption happen entirely inside your browser using the Web Crypto API. The only network request VaultLock ever makes is the initial page load. Your files, keys, and vault outputs never leave your device.
What is AES-256-GCM? +
AES-256-GCM is the industry standard for authenticated symmetric encryption, standardized by NIST (FIPS 197, SP 800-38D). It provides both confidentiality (no one can read your file without the key) and integrity (any modification to the encrypted file is detected before decryption).
What happens if I lose my password? +
The password cannot be recovered — it is never stored anywhere. This is a deliberate security property: there is no server that could be breached to reveal your key. Store your password in a password manager or a secure notes app immediately after copying it.
Can I encrypt multiple files at once? +
Yes. Select as many files as you like — VaultLock will bundle them all into a single ZIP archive using DEFLATE compression before encrypting. You receive one .vault file regardless of how many files you added. On decryption, all original files are restored individually.
What is the .vault file format? +
A .vault file contains three sections: an ASCII header identifying the file format and version, a 96-bit random initialization vector (IV) encoded as hex, and the AES-256-GCM ciphertext including its 128-bit authentication tag. The format is self-contained and portable.
Does it work offline? +
Yes. Once the page has loaded, VaultLock performs all operations using browser-native APIs and the pre-loaded JSZip library. You can disconnect from the internet and continue encrypting or decrypting files. No additional network requests are made during any operation.
